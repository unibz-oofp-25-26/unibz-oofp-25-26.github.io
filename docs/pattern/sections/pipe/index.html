    Pipeline #

    Motivation #

    Syntax. Consider a chain of function applications, where the output of each function is the (unique) input to the next function in the chain. For instance

    $\qquad h(g(f(x)))$

    Computationally, this reads from right to left :

    $\qquad$ “start with $x$, then apply $f$ to it, then apply $g$ to the result, and then $h$ to the result”.

    Instead, when we write code, it can be convenient to use an expression that mimics the order of execution (when read from left to right).

    For instance, we could define a reverse application symbol $\vartriangleright$ and write:

    $\qquad ((x \vartriangleright f) \vartriangleright g) \vartriangleright h$

    Or, assuming that $\vartriangleright$ associates to the left:

    $\qquad x \vartriangleright f \vartriangleright g \vartriangleright h$

    Note. A similar observation can be made for function composition: $g \circ f$ means “$f$ followed by $g$”. Instead, it can be more convenient to write $f$ before $g$.

    Usage #

    In functional languages #

    Many functional languages provide operators for reverse application and/or composition, or let us define such operators (as higher-order functions).

    These are often used to specify a sequence of operations (filter, transform, aggregate, etc.) applied to a stream of incoming data. This is analogous to a sequence of Unix-like pipes, where the output of a command is fed as input to the next command.

    Thanks to lazy evaluation, each operation in the pipeline can start feeding partial results to the next one, even if it has not finished processing its input. This style of programming also pairs well with functors and monads, which we will see in later sections.

    Beyond functional languages #

    A variety of imperative languages (notably C++, C#, Java or Rust) have borrowed or adapted this pattern. For instance, pipelines have become an essential part of Java programming, since the release of Java Stream API (2014).

    This pattern has also made its way to scripting languages for statistical analysis, such as R (natively) or Matlab (via libraries).

    in Haskell #

    Syntax. The Haskell module Data.Function provides a reverse application function called & (equivalent to the $\vartriangleright$ symbol that we used above). It is defined as expected:

    (&) :: a -> (a -> b) -> b
x & f = f x

    This function associates to the left, i.e. x & f & g means (x & f) & g

    Example.

    h (g (f x))

    can be written

    x & f & g & h

    Or more commonly:

    x & f
  & g
  & h

    Recall the following functions:

    • even :: Int -> Bool, which maps an integer to True iff it is even,
    • filter :: (a -> Bool) -> [a] -> [a], where filter f xs retains the elements of xs for which f evaluates to True,
    • map :: (a -> b) -> [a] -> [b], where map f xs applies f to each elements of xs,
    • all :: (a -> Bool) -> [a] -> Bool, where all f xs is True iff f evaluates to True for every element in xs
    • maybeFirst :: [a] -> Maybe a, which maps a nonempty list its head (and the empty list to Nothing),
    • maybeLast :: [a] -> Maybe a, which maps a nonempty list its last element (and the empty list to Nothing).

    Evaluate the following expressions.

    1. [2,1,4,7] & filter even
          & map (+1)
          & maybeLast

    2. [-5,4,2,-1] & filter (\x -> x >= 0)
            & map (\x -> -x)
            & filter (\x -> x > -2)
            & all (not even)

    3.    [ ("Bolzano", "BZ", 39100),
     ("Rovereto", "TN", 38068),
     ("Merano", "BZ", 39012),
     ("Bressanone", "BZ", 39042) ] & filter \(_, "BZ", zip) -> zip /= 39100
                                   & map \(name, _, _) -> name
                                   & maybeFirst
    1. Just 5
    2. True
    3. Just "Merano"

    Simplify the following function.

    myPipe :: [Int] -> Maybe Int
myPipe xs = xs & filter even
               & map (+1)
               & filter even
               & maybeFirst

    myPipe :: [Int] -> Maybe Int
myPipe _ = Nothing
