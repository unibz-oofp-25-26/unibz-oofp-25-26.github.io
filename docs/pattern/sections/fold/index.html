    Fold #

    A variety of operations on a list can be understood as combining its values inductively, either from left to right or from right to left.

    Example. Consider the list

    $\qquad 1, 2, 3, 4$

    The sum of its elements can be computed inductively as:

    • $((1 + 2) + 3) + 4\qquad$ (left-to-right), or
    • $1 + (2 + (3 + 4))\qquad$ (right-to-left)

    Similarly, the least element in this list can be computed as

    • $\min(\min(\min(1, 2), 3), 4)\qquad$ (left-to-right), or
    • $\min(1, \min(2, \min(3, 4)))\qquad$ (right-to-left)

    Note. At first sight, these expressions may look different for + and $\min$. The only reason is that + is traditionally written in infix form, whereas $\min$ is more often written in prefix form. To convince yourself that these expressions have the same structure, you may for instance replace each $(a + b)$ in the first expression with $\mathsf{sum} (a,b)$.

    Generalization. This naturally leads to defining two higher-order functions (left-to-right and right-to-left) that take as argument a binary operation $\oplus$ (e.g. + or $\min$), and apply it inductively to a list:

    • $((1 \oplus 2) \oplus 3) \oplus 4\qquad$ (left-to-right), or
    • $1 \oplus (2 \oplus (3 \oplus 4))\qquad$ (right-to-left)

    Warning. $+$ is an associative operator, meaning that

    $\qquad(a + b) + c = b + (a + c)$

    And similarly for $\min$.

    For these operators, left-to-right and right-to-left evaluation coincide. In contrast, for a non-associative operation (like $-$, $/$, set difference, etc.), these may produce different results.

    Example.

    $\qquad(1 - 2) - 3 = - 4$,

    whereas

    $\qquad1 - (2 - 3) = 0$

    Terminology. In functional programming, combining the values of a recursive data structure (like a list here) is often called a fold (or sometimes a reduce operation).

    The left-to-right strategy is a left fold, and the right-to-left strategy a right fold.

    Note. A right fold on a list mimics its recursive structure. For that reason, right folds are sometimes easier to understand than left folds.

    Note. Folds are not limited to lists, but apply to other recursive structures, notably trees.

    in Haskell #

    Consider the two functions below:

    or :: [Bool] -> Bool
or = foldr (||) False

myFunction :: (a -> Bool) -> [a] -> Bool
myFunction f = or . map f
    1. Evaluate the following expressions:
    • myFunction (\x -> x > 2) [-2, 0, 2]
    • myFunction (\x -> x > 2) [3, 0, 2]
    • myFunction (\x -> x /= 5) [3, 0]
    • myFunction (\x -> x /= 5) [5]
    • myFunction (\x -> x /= 5) []

    What does the function myFunction do?

    • myFunction (\x -> x > 2) [-2, 0, 2] evaluates to False
    • myFunction (\x -> x > 2) [3, 0, 2] evaluates to True
    • myFunction (\x -> x /= 5) [3, 0] evaluates to True
    • myFunction (\x -> x /= 5) [5] evaluates to True
    • myFunction (\x -> x /= 5) [] evaluates to False
    1. This function is equivalent to the function any that we have seen already.
