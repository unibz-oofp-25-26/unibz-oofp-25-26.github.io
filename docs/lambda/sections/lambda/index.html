    Lambda expression #

    In certain programming languages like Java, C#, C++ or JavaScript, the term lambda expression is used to denote an anonymous method with a certain syntax.

    The term is a (loose) reference to lambda calculus, a model of computation exclusively based on function composition and application.

    Syntax in Java #

    A Java method

    <ReturnType> myMethod (<Type 1> <argument 1>, .., <Type n> <argument n>){ <Body> }

    can be written anonymously as the lambda expression:

    (<Type 1> <argument 1>, .., <Type n> <argument n>) -> { <Body> }

    For instance, the method:

    int division (int x, int y){
    return x / y;
}

    is equivalent to the lambda expression

    (int x, int y) -> { return x / y ; }

    However, a more concise syntax can often be used:

    Arguments #

    If the types of the arguments can be inferred by the compiler (which is often the case), then they can be omitted.

    Example.

    (int x, int y) -> { ... }

    may be written

    (x, y) -> { ... }

    If the method has exactly one argument and its type is omitted, then parentheses can also be omitted.

    Example.

    (x) -> { ... }

    can always be written

    x -> { ... }

    If the method takes no argument, then parentheses are mandatory.

    Example.

    () -> { ... }

    Body #

    If:

    • the body of a lambda expression consists of a single statement, and
    • the return type of this lambda expression (or void if there is none) can be inferred by the compiler (which is often the case),

    then the semicolon (;), curly braces ({ and }) and return keyword (if any) can be omitted.

    Example.

    (x, y) -> {
    return x / y ;
}

    can often be written

    (x, y) -> x / y
