    Function #

    Function #

    Syntax. A function in Haskell is defined analogously to a mathematical function, with the symbol =.

    Example. The function $f(x) = 2x$ is written in Haskell

    f x = 2 * x

    Syntax. Note that $f(x)$ is written f x, without parentheses. This also holds when applying a function to a constant.

    Example. $f(15)$ is normally written

    f 15

    Function type #

    Syntax. The type of a Haskell function is specified immediately above its definition, as follows

    <functionName>:: <type>

    Example. The function $\textit{square}\colon \mathbb{Z} \to \mathbb{Z}$ defined by $\textit{square}(x) = x^2$ is written

    square:: Integer -> Integer
square x = x^2

    Example. The following function maps even integers to True, and odd ones to False. It has type Integer -> Bool

    even:: Integer -> Bool
even x = x % 2 == 0

    In this definition, % is the so-called “modulo” operator: a % b is the remainder of the integer division of a by b.

    Function types in Haskell are optional, but highly recommended.

    Hint. Function types in Haskell can be automatically inferred.

    So you may use them in two alternative ways:

    1. For a simple function:
    • implement the function,
    • then use your IDE to generate a type for it.
    1. For a more complex function:
    • write the function type,
    • then implement the function,
    • use your IDE to check that your implementation meets the type requirements.

    Note that the first option option may produce a type that is too generic for your needs.

    Currying #

    A function can take a tuple as argument.

    Example.

    exp:: (Integer, Integer) -> Integer
exp (x,y) = x^y

    However, functions in curried form are usually preferred, because they are more versatile.

    Example. The function above in curried form is

    expCurried:: Integer -> (Integer -> Integer)
(expCurried x) y = x^y

    Syntax. Parentheses in the above example are redundant, because:

    • the -> operator for types associates to the right, and
    • function application associates to the left.

    So this function is normally written:

    expCurried:: Integer -> Integer -> Integer
expCurried x y = x^y

    Write in Haskell the function $apply(f,x)$ in curried form, where:

    • $f$ is a function that maps strings to Boolean values,
    • $x$ is a string,
    • $apply(f,x) = f(x)$.
    apply:: (String -> Bool) -> String -> Bool
apply f x = f x

    Write in Haskell the curried form of the function $applySwapped(x,f)$, which is identical to the function above, but where $f$ and $x$ are swapped.

    applySwapped:: String -> (String -> Bool) -> Bool
applySwapped x f = f x

    Consider the following Haskell program, where the function unicodeCodePoint:: Char -> Int maps a character to its Unicode number.

    aFunction i j f c = anotherFunction i j (f c)

anotherFunction:: Int -> Int -> Int -> Int
anotherFunction i j k = (i * j) + k

main = print $ aFunction 10 3 unicodeCodePoint 'V'

    • Int -> Int -> Char -> (Int -> Char) -> Int
    • (Int -> Int) -> (Char -> Int) -> Char -> Int

    • (Int -> Int) -> (Char -> Int -> Char) -> Int

    Infix notation #

    Some native Haskell functions are written in infix notation. These are often binary operators, like + (sum of two numbers) or && (Boolean conjunction). These function are implicitly in curried form.

    Example. The function && has type Bool -> Bool -> Bool.

    Example. 2 - is the function that subtracts a number from two.

    In other words, the two functions below are equivalent:

    myFunction:: Integer -> Integer
myFunction x = 2 - x

theSameFunction:: Integer -> Integer
theSameFunction = 2 -

    Surprisingly, an infix function in Haskell can also be called with its “second argument” only (whereas a prefix function cannot).

    Example. - 2 is the function that subtracts two from a number.

    In other words, the two functions below are equivalent:

    myFunction:: Integer -> Integer
myFunction x = x - 2

theSameFunction:: Integer -> Integer
theSameFunction = - 2

    Infix to prefix #

    An infix function can be used as a regular (i.e. prefix) one, by surrounding it with parentheses.

    Examples.

    infix form equivalent prefix form
    2 + 3 (+) 2 3
    True && True (&&) True True

    Example. The two functions below are equivalent:

    myFunction:: Integer -> Integer
myFunction = 2 -

theSameFunction:: Integer -> Integer
theSameFunction = (-) 2

    This is helpful in contexts where the infix form cannot be used.

    Example. We can pass the function && to another function, by surrounding it with parentheses.

    applyBoolOp:: (Bool -> Bool -> Bool) -> Bool -> Bool -> Bool
applyBoolOp f x y = f x y

main = print $ applyBoolOp (&&) True False

    We can also define an infix function. In this case, the type declaration must use parentheses.

    Example. The program below define a function myOp in infix form.

    (myOp):: Integer -> Integer -> Bool
x myOp y = x > 0 && x < y

main = print $ 2 myOp 4

    Prefix to infix #

    Conversely, a curried function in prefix form can be used as an infix one, by surrounding it with `.

    Examples.

    prefix form equivalent infix form
    exp 2 3 2 `exp` 3
    isPrefixOf "foo" "foobar" "foo" `isPrefixOf` "foobar"

    This allows us to call a function in curried form with its “second argument” only.

    Example. The two functions below are equivalent.

    myFunction:: String -> Bool
myFunction x = isPrefixOf x "fooBar"

theSameFunction:: String -> Bool
theSameFunction = `isPrefixOf` "fooBar"
