    Sum type #

    A sum type represents several alternative types.

    in Haskell #

    Syntax. A sum type in Haskell is declared as

    data <sum type name> = <first type definition> | ... | <last type definition>

    Example. In the following declarations:

    • Point is a product type, and
    • Shape is a sum type.
    -- a point in the Euclidean plane, represented by its coordinates
data Point = Point Double Double

-- a shape, defined as either a triangle or a rectangle
data Shape = Triangle Point Point Point | Rectangle Point Point Point Point

    Note. in Haskell, an enumerated type is a sum type.

    More precisely, the alternative “values” of an enumerated type are nullary (a.k.a. 0-ary) constructors, i.e. constructors with no argument.

    Pattern matching #

    Just like an enumerated type, a sum type provides alternative patterns.

    Example (continued). Let us define a function perimeter that maps a Shape to its perimeter.

    First, we define an auxiliary function distance (in curried form) that maps two points to their distance:

    distance :: Point -> Point -> Double
distance (Point x1 y1) (Point x2 y2) = sqrt ( (x2 - x1) ^ 2 + (y2 - y1) ^ 2 )

    In this definition, observe the usage of the constructor Point to create the two patterns Point x1 y1 and Point x2 y2.

    We can now define our main function:

    perimeter :: Shape -> Double
perimeter Triangle p1 p2 p3 = distance p1 p2 + distance p2 p3 + distance p1 p3
perimeter Rectangle p1 p2 p3 p4 = 2 * (distance p1 p2 + distance p2 p3)

    Here we created one pattern out of each alternative definition of the sum type Shape:

    • the pattern Triangle p1 p2 p3, and
    • the pattern Rectangle p1 p2 p3 p4.
