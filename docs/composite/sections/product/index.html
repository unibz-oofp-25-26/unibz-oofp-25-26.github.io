    Product type #

    A product type can be viewed as a simplistic class, and its instances as simplistic objects.

    in Haskell #

    Syntax. Like an enumerated type, a product type in Haskell can be declared with the keyword data, as follows

    data <product type name> = <definition>

    There are two alternative syntaxes for this <definition>:

    • a tuple type, or
    • a definition with a constructor.

    Definition as a tuple type #

    Example. The declaration below defines a product type Interval.

    -- a temporal interval, represented as two timestamps (start and end)
data Interval = (Int, Int)

    In this definition, (Int, Int) is the type of pairs of integers.

    Example. The expression below define a product type Signal.

    A Signal consists of:

    • an Interval, and
    • a value for the enumerated type Light seen earlier.
    -- a traffic signalling event, represented as a temporal interval and the light (Green, Yellow or Red) during that interval
data Signal = (Interval, Light)

    In this definition, (Interval, Light) is the type of all pairs with:

    • a first element of type Interval, and
    • a second element of type Light.

    Instances #

    Syntax (reminder). A tuple in Haskell is written in parentheses.

    Examples. Consider the two product types that we just defined:

    data Interval = (Int, Int)
data Signal = (Interval, Light)

    Then:

    • the pair (5,8) has type Interval,
    • the pair ((5,8), Red) has type Signal.

    Definition with a constructor #

    Haskell provides another syntax to define a product type, by means of a constructor. As we will see later, when combined with sum types, this syntax allows:

    • concise pattern matching, and
    • recursive types.

    So this is often the preferred way to define a product type.

    Syntax. A product type with constructor can be declared as

    data <product type name> = <constructor> <type name 1> ... <type name n>

    where the constructor and each type name starts with a capital letter.

    Example (continued). The two types above can alternatively be declared:

    data Interval = IntervalC Int Int
data Signal = SignalC Interval Light

    The constructors for these types are IntervalC and SignalC

    Syntax. The name of a constructor is arbitrary.

    Example. the type Interval can equivalently be declared

    data Interval = MyConstructor Int Int

    Warning. The name of a product type and its constructor can be identical (as in Java).

    This is a widely adopted practice in Haskell.

    Example.

    data Interval = Interval Int Int
data Signal = Signal Interval Light     -- `Interval` here stands for the type, not the constructor

    Convention. In this course, for didactic purposes, we will different names for a product type and its constructor.

    Instance #

    An instance of such a type is created with its constructor (as in Java).

    Examples. Consider the two product types that we have just defined:

    data Interval = IntervalC Int Int
data Signal = SignalC Interval Light

    Then:

    • the expression IntervalC 5 10 has type Interval,

    • the expression SignalC (Interval 5 10) Red has type Signal.

    Pattern matching #

    A constructor can be used to create complex patterns.

    Example. In the program below:

    • the function getLight maps a signal to its light (i.e. Green, Yellow or Red),
    • the function isLenghty maps a signal to True if its interval is longer than 3.
    data Interval = IntervalC Int Int
data Signal = SignalC Interval Light

getLight :: Signal -> Light
getLight SignalC _ l = l

isLenghty :: Signal -> Bool
isLenghty SignalC (IntervalC b e) _ = e - b > 3

    Consider the following program:

    data Interval = IntervalC Int Int
data Signal = SignalC Interval Light

aFunction :: [Signal] -> [Interval]
aFunction [] = []
aFunction (SignalC i Red : ss) = i : aFunction ss
aFunction (_ : ss) = aFunction ss
    1. What does the following expression evaluate to?
    aFunction [
  SignalC (IntervalC 0 2) Red,
  SignalC (IntervalC 2 3) Green,
  SignalC (IntervalC 3 4) Orange,
  SignalC (IntervalC 4 6) Red,
]
    1. More generally, what does the function aFunction do?
    [(IntervalC 0,2), (IntervalC 4,6)]
    1. It maps a list of signals to the intervals for which the light was red.

    Same two questions as above, for the function anotherFunction below (recall the higher-order functions filter and map seen earlier).

    data Interval = IntervalC Int Int
data Signal = SignalC Interval Light

anotherFunction :: [Signal] -> [Interval]
anotherFunction = (map \(SignalC i _) -> i) . (filter \(SignalC _ l) -> l == Red)

    It is equivalent to the function of the previous exercise.

    Unary constructor #

    Syntax. A constructor can also create “tuples” that consist of a single value. This will be useful to define recursive types.

    Example. The following type is valid in Haskell.

    data DummyType = DummyTypeC Int

    Record-like notation #

    Syntax. Haskell provides an alternative notation for a product type with constructor, using attribute names. This is very similar to structures in C or records in Java. For instance:

    data Interval = Interval { start :: Int, end :: Int }
data Signal = Signal { interval :: Interval, light :: Light }
