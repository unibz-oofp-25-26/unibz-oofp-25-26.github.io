    Enumerated type #

    An enumerated type is a finite set of possible values. This is a more readable alternative to so-called “magic numbers”.

    Example. A traffic light can have three states: red, orange and green.

    We could model this with a “magic number”. For instance, a variable x with type integer, where:

    • x < 0 means red,
    • x == 0 means orange,
    • x > 0 means green.

    Instead, we can declare an enumerated type TLState with 3 possible values (Red, Orange and Green), and use a variable with this type. Then instead of

    x < 0,

    we can write

    x == Red,

    which is more readable.

    In Haskell #

    Syntax. An enumerated type in Haskell can be written

    data <enumerated type name> = <first value > | ... | <last value>

    where the type name and the values start with a capital letter.

    Example.

    data TLState = Red | Orange | Green

    Note. The type Bool in Haskell is an enumerated type (with values True and False).

    An enumerated type can be used like a regular one (e.g. in a pattern or a Boolean condition).

    Example.

    data TLState = Red | Orange | Green

-- pattern matching
canPass :: TLState -> Bool
canPass Red = False
canPass _ = True

-- Boolean condition
isRisky :: Int -> TLState -> Bool
isRisky speed state = speed > 30 && state != Green
