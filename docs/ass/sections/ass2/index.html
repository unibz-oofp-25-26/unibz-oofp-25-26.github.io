    Exercise 1: utility functions #

    lastMatch #

    Implement the function

    lastMatch :: (a -> Bool) -> [a] -> Maybe a

    such that lastMatch f xs is:

    • Just x if x is the last element of the list xs for which f x is True, or
    • Nothing if there is no such x.

    For instance,

    lastMatch even [2,3,4,2,5]

    should evaluate to Just 2, whereas

    lastMatch even [3,5]

    should evaluate to Nothing.

    lastMatch :: (a -> Bool) -> [a] -> Maybe a
lastMatch f xs = find f (reverse xs)

    firstAgreement #

    Implement the function

    firstAgreement :: (a -> b) -> (a -> b) -> [a] -> Maybe b

    such that

    firstAgreement f g xs is:

    • Just (f x) if x is the first element of the list xs for which f x is equal to g x, or
    • Nothing if there is no such x .
    firstAgreement :: (a -> b) -> (a -> b) -> [b] -> Maybe b
firstAgreement f g xs = xs
              & find (\x -> (f x == g x))
              & fmap f

    For instance,

    firstAgreement (+ 2) (* 2) [5,2]

    should evaluate to Just True,

    whereas

    firstAgreement (even) (> 3) [3]

    should evaluate to Nothing.

    compose #

    Terminology. If

    $\qquad F = f_1, f_2, \ldots, f_n$

    is a list of functions with $f_i \colon X \to X$ for each $f_i$, then we call inductive composition of $F$ the function

    $\qquad f_1 \circ f_2 \circ \ldots \circ f_n$

    Note. Function composition is associative, so no parenthesis is needed here.

    Implement the function

    compose :: [a -> a] -> a -> a

    such that compose fs is:

    • the inductive composition of all functions in fs if fs is nonempty, or
    • the identity function otherwise.

    For instance,

    • compose [(*3), (+2)] is the function \x -> (x + 2) * 3
    • compose [(+2)] is the function (+2)
    • compose [] is the identity function id

    Hint. You may take advantage of one of the native higher-order functions that we have encountered.

    keepLastOccIn #

    Implement the function

    keepLastOcc :: (Eq a) => a -> [a] -> [a]

    such that keepLastOccIn x xs is the list identical to xs for all elements different from x, and where only the last occurrence of x (if any) is retained.

    For instance:

    • keepLastOccIn 2 [1,2,3,2,4,5,2,4] should evaluate to [1,3,4,5,2,4]
    • keepLastOccIn 2 [1,3,4] should evaluate to [1,3,4]
    • keepLastOccIn 2 [2] should evaluate to [2]
    • keepLastOccIn 2 [] should evaluate to []

    Hint. You may use recursion here, with either:

    • additional intermediate results, or
    • an accumulator.
    keepLastOcc :: a -> [a] -> [a]
keepLastOcc x xs = ys
    where (_, ys) = keepLastOccRec x xs

keepLastOccRec :: a -> [a] -> (Bool, [a])
keepLastOccRec _ [] = []
keepLastOccRec x (h : xs) | h == x && not b = (True, h : xs)
                          | otherwise = bm xs
      where (b, ys) = keepLastOcc x (h : xs)

    discardDups #

    Implement the function

    discardDups :: (Eq a) => [a] -> [a]

    such that discardDups xs is the list identical to xs, but where only the last occurrence of each element of xs is retained.

    discardDups :: [a] -> [a]
discardDups [] = []
discardDups (x : xs) = keepLastOcc x (discardDups xs)

    Exercise 2: tracing a virus #

    Data format. We will use the following product types to represent infection tests for a virus, as well as meetings:

    data Person = PersonC {
    name :: String,
    city :: String
} deriving (Show, Eq)

data Test = TestC {
    subject :: Person,
    time :: Int,
    positive :: Bool
} deriving (Show, Eq)

data Meeting = MeetingC {
    time :: Int,
    attendees :: [Person]
} deriving (Show, Eq)

    The time attributes are integers that identify days, numbered from a certain origin.

    We assume for simplicity that all people have different names.

    Running example. We will use the following sample data to illustrate below the functions that we want to implement:

    alice, bob, carol :: Person
alice = PersonC { name = "Alice", city = "Bolzano" }
bob = PersonC { name = "Bob", city = "Bolzano" }
carol = PersonC { name = "Carol", city = "Trento" }

test1, test2 :: Test
test1 = TestC { subject = alice, time = 30, positive = False }
test2 = TestC { subject = alice, time = 42, positive = True }

m1, m2 :: Meeting
m1 = MeetingC { time = 46, attendees = [alice, bob] }
m2 = MeetingC { time = 58, attendees = [alice, carol] }

    Implement the functions below by completing the function stubs in the file Src/Virus.hs.

    Hint. Some of these functions can be used to implement others.

    Functions of the previous exercise can be used as well.

    Hint. Feel free to create additional (auxiliary) functions to simplify your implementation.

    neighborsOf #

    Implement the function

    neighborsOf :: Person -> [Person] -> [Person]

    such that neighborsOf p ps is the list of people in ps who live in the same city as p, excluding p, and in the same order as they appear in ps.

    Restriction. You can assume that the list ps does not contain duplicates (but it may contain p).

    neighborsOf :: Person -> [Person] -> [Person]
neighborsOf p ps = [p' | p' <- ps, city p == city p', p /= p']

    attendances #

    Implement the function

    attendances :: Person -> [Meeting] -> [Meeting]

    such that attendances p ms is the list of all meetings in ms that p attended, in the same order as they appear in ms.

    attendances :: Person -> [Meeting] -> [Meeting]
attendances p ms = [ m | m <- ms, any (== p) (attendees m) ]

    metDuring #

    Implement the function

    metDuring :: Person -> Person -> Meeting -> Bool

    such that metDuring p1 p2 m is True iff p1 and p2 are different people and both attended meeting m.

    metDuring :: Person -> Person -> Meeting -> Boolean
metDuring p1 p2 m = any (== p1) (attendees m) &&
  any any (== p2) (attendees m) &&
  p1 /= p2

    lastMeetingBefore #

    Implement the function lastMeetingBefore :: Person -> Person -> Int -> [Meeting] -> Maybe meeting

    such that

    lastMeetingBefore p1 p2 t ms is the last meeting that started before t where p1 and p2 met, or Nothing is there is no such meeting.

    You can assume that the input meetings are sorted in chronological order.

    lastMeetingBefore :: Person -> Person -> Int -> [Meeting] -> Maybe meeting
lastMeetingBefore p1 p2 t ms = reverse ms & find (metDuring p1 p2)

    Implement the function firstPositiveTest:: Person -> [Test] => Maybe Test

    such that

    firstPositiveTest p ts is the first positive test in ts whose test subject was p, if any.

    Restriction. You can assume that tests in ts are in chronological order.

    possibleTransmitter #

    possibleTransmitter p1 p2 ts ms evaluate to true True iff:

    • p2 tested positive, and
    • p2 met p1 at most a week before the last positive test of p2.

    Hint. You may take advantage here of:

    • the functions firstPositiveTest and lastMeetingBefore implemented above, and

    contactsSince #

    Write a function

    contactsSince :: Person -> Int -> [Meeting] -> [String]

    such that

    contactsSince p t ms

    is the list of names of people that p met in any meeting in ms that took place at a time >= t. The order of names in the output list does not matter. This list:

    • should not contain duplicates, and
    • should not contain the name of p.
    contactsSince p t ms = ms & filter (data >= t)
                          & filter
