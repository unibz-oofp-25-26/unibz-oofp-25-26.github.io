    Exercise 1: algebraic expressions #

    Conventions. In this exercise, an algebraic expression is either:

    • an integer, or
    • an operation, i.e. an operator (+, * or /) with two operands.

    The operator / stands for integer division, meaning that the result is rounded down.

    Example. The expression 7 / 3 evaluates to 2.

    We represent an algebraic expression as an instance of the Java interface Expression in the following diagram:

    Example. The expression

    (3 * 6) + ((7 + 3) / 2)

    may be viewed as the following algebraic tree:

    In this tree:

    • the two expressions with a + as root are instances of the class Sum,
    • the expression with a * as root is an instance of the class Product,
    • the expression with a / as root is an instance of the class Division,
    • the atomic expressions are instances of the class IntegerValue.

    Complete the classes IntegerValue, Sum, Product, Division and AbstractOperation so that the non-abstract classes correctly implement the interface Expression. The expected behavior of each method is described in the interface.

    Hint. Feel free to:

    • create additional methods, including recursive ones, and/or
    • use the abstract class to avoid duplicate code.

    Hint. You can debug your code by running the tests in the folder test/java/ex1.

    Exercise 3: recommender system #

    This exercise simulates a simplified recommender system for a movie streaming platform.

    Data model.

    • Each User has an id and a watching history, which is a list of movies, in the order in which they were watched. The system only keeps track of the first time a movie was watched (therefore a movie cannot appear twice in this list).

    • A Movie has an id, a title, a genre, a release year and a set of reviews.

    • A Review has an id and a score (an integer from 0 to 5).

    Warning. These three classes should not be modified.

    Note. A simpler implementation would use Java records for these three types, rather than classes (but records are not supported by Codeboard).

    Complete the class PipelineRecommender so that it correctly implements the interface Recommender.

    Hint. You can use Java streams to simplify your code.

    Hint. Feel free to create auxiliary methods to simplify your streams (or loops if you did not use streams).

    Hint. You may use some methods of Recommender to implement others.

    Hint. You can debug your code by running the tests in the folder test/java/ex3.
