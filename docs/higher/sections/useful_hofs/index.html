    Some higher-order functions #

    We introduce a few convenient higher-order functions.

    In Haskell #

    filter #

    The function filter retains elements of a list that satisfy a certain condition.

    It can be defined as follows:

    filter :: (a -> Bool) -> [a] -> [a]
filter f xs  = [x | x <- xs, f x]

    Example. Recall the function even that we defined earlier, as follows:

    even :: Integer -> Boolean
even x = x % 2 == 0

    Then the expression

    filter even [1,4,6,4]

    evaluates to

    [4,6,4]

    Equivalently, instead of defining the function even, we could inline it (as an anonymous function) in the expression above, i.e. we could write

    filter (\x -> x % 2 == 0) [1,4,6,4]

    Implement the function

    retainIfEqual :: (Int -> Int) -> Int -> [Int] -> [Int]

    such that retainIfEqual f y xs consists of each element x of xs for which f x is equal to y (in the same order as they appear in xs).

    For instance,

    retainIfEqual (\x -> x^2) 4 [2,0,-2,-1,2]

    should evaluate to

    [2,-2,2]

    retainIfEqual :: (Int -> Int) -> Int -> [Int] -> [Int]
retainIfEqual f y = filter (\x -> x == f y)

    map #

    The function map applies a function to every element of a list.

    It is defined as follows:

    map :: (a -> b) -> [a] -> [b]
map f xs  =  [f x | x <- xs]

    Examples.

    • map (+1) [1,4,5,4] evaluates to [2,5,6,5]
    • map even [1,4,5,4] evaluates to [False, True, False, True]
    • map (\x -> x % 2 == 0) [1,4,5,4] is equivalent

    Implement the function

    maxes :: [(Int,Int)] -> [Int]

    that retains the smaller element in each pair of the input list.

    For instance,

    mins [(0,3),(2,-1),(3,3)]

    should evaluate to

    mins [0,-1,3]

    mins :: [(Int,Int)] -> [Int]
mins = map min

min:: Int -> Int -> Int
min x y | x <= y = x
        | otherwise = y

    mins :: [(Int,Int)] -> [Int]
mins = map (\(x,y) | x <= y -> x
                   | otherwise -> y)

    all and any #

    The functions all and any have type (a -> Bool) -> [a] -> Bool.

    • The expression all f xs evaluates to true iff f x evaluates to true for every element x of the list xs.
    • The expression any f xs evaluates to true iff f x evaluates to true for at least one element x of the list xs.

    Note. We will see how these two functions can be implemented in the section on recursion.

    Examples.

    • all even [1,4,5,4] evaluates to False

    • all even [4,4,2] evaluates to True

    • all even [] evaluates to True

    • any even [1,4,5,2] evaluates to True

    • any even [1,3] evaluates to False

    • any even [] evaluates to False

    Exercise.

    • implement the function contains :: Eq a => a -> [a] -> Bool

    such that contains x xs is True iff x appears in xs

    Possible solution.

    contains :: Eq a => a -> [a] - Bool
contains x xs = any (\y -> x == y) xs

    Exercise.

    • implement the function restrictTo :: Eq a => [a] -> [a] -> [a]

    such that restrictTo xs ys consists of all elements of xs that appear in ys, in the same order as they appear in xs (and preserving duplicates).

    Possible solution.

    restrictTo :: Eq a => [a] -> [a] -> [a]
-- reuse the function contains defined in the previous exercise
restrictTo xs ys = filter (\x -> contains x ys)

    Alternative solution.

    contains :: Eq a => [a] -> [a] -> [a]
-- without auxiliary function
contains xs ys = filter (\x -> any (\y -> y == x) ys) xs

    flip #

    Th function flip intuitively swaps the arguments of a function with two arguments in curried form:

    It is defined as follows:

    flip :: (a -> b -> c) -> b -> a -> c
flip f y x = f x y

    Example

    -- Checks whether an element x appears in a list xs
isIn :: Eq a => a -> [a] -> Bool
isIn x xs = anyMatch (\y -> y == x) xs


-- Checks whether a list xs contains an element x
contains :: Eq a => [a] -> a -> Bool
contains = flip isIn
