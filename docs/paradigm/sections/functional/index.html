    Functional programming #

    Side effect #

    Definition. A method/function has a side effect if it modifies a resource (variable, object, array, etc.) that is defined outside of its scope.

    Pure function #

    A pure function (or method) essentially behaves like a mathematical function. More precisely:

    Definition. A function is pure if it:

    1. returns the same value if called twice with the same argument(s),
    2. has no side-effect.

    Example #

    Example. Consider the following Java class:

    public class MyClass {

    int x = 0;

    ...

    boolean inc() {
        x++;
        return true;
    }

    boolean isEqual(int y) {
        return x == y;
    }
}
    • inc is not pure, because it violates Condition 2 (side effect).
    • isEqual is not pure either, because it violates Condition 1. For instance, if:
      • x has value 0,
      • then isEqual(0) is executed,
      • then inc() is executed,
      • then isEqual(0) is executed again.

    Benefits #

    Easier debugging #

    Because it has no side effect, a pure function may cause a bug only if it returns an incorrect value.

    Readability #

    The behavior of a pure function can be fully documented by describing its signature (arguments and return value).

    Parallelization #

    Example. If $f$ and $g$ are pure functions, then we can evaluate

    $\qquad$ f(a) + g(b)

    by executing f(a) and g(b) in parallel.
