    Conditions #

    Pattern matching #

    A pattern is a syntactic expression that can be matched against certain values.

    Examples. Here are some basic patterns:

    • the pattern 2 matches the value 2,
    • the pattern True matches the value True,
    • a variable name (like x, y or xs) matches anything.

    Patterns let us define a function on a case-by-case basis.

    Example. The function below maps 0 to 0, and every other integer to its successor. It uses the patterns 0 and x.

    fancySucc:: Integer -> Integer
fancySucc 0 = 0
fancySucc x = x + 1

    Warning. The first pattern that matches a value takes precedence. So the order of patterns matters.

    Exercise

    What does the following function do? Can it be simplified?

    dummySucc:: Integer -> Integer
dummySucc x = x + 1
dummySucc 0 = 0

    Solution.

    It maps an integer to its successor. The second case is useless, because the input 0 is already handled by the first pattern. So the function is equivalent to

    succ:: Integer -> Integer
succ x = x + 1

    Patterns can be composed into tuples

    Example.

    The following function maps a pairs of Boolean values to their conjunction.

    and:: (Bool, Bool) -> Bool
and (True,True) = True
and (x,y) = False

    Exercise

    Write the function above in curried form, and then in (curried) infix form:

    Solution

    Curried form.

    and:: Bool -> Bool -> Bool
and True True = True
and x y = False

    Infix form.

    (and):: Bool -> Bool -> Bool
True and True = True
x and y = False

    Exercise

    What does the following function do?

    (op):: Bool -> Bool -> Bool
False op x = False
True op x = x

    It is equivalent to the function and above.

    Warning

    A same variable name cannot be used twice on the left hand side of an equation.

    Example.

    The following is not valid syntactically in Haskell

    (invalidXNor):: Bool -> Bool -> Bool
x invalidXNor x = True
x invalidXNor y = False

    The null pattern #

    The null pattern _ matches any value (like a variable name), but the value that it matches cannot be reused. It is convenient to indicate that the value is irrelevant.

    Example

    The function (op) above can be written

    (op):: Bool -> Bool -> Bool
False op _ = False
_ op x = x

    Contrary to a variable, the null pattern can be used several times on the left hand side of an equation. In such an equation, the values matched by _ can be different.

    Example.

    The function (and) above can be written

    (and):: Bool -> Bool -> Bool
True and True = True
_ and _ = False

    Complex patterns #

    We will see later on that pattern matching extends to lists and composite types.

    Boolean expression #

    Pattern matching has limitations. For instance, no pattern can match integers greater than 2, or odd integers.

    For such conditions, we can use Boolean expressions

    We have seen above how to define a Boolean operator. But this is not necessary: Haskell provides native implementations, which are similar their counterparts in Java (and most programming languages). They include:

    • conjunction, noted &&
    • disjunction, noted ||
    • negation, noted not

    Examples.

    • 3 < 2 && 3 > 5 evaluates to False
    • not (3 > 5 || True) evaluates to False

    Guards #

    A guard is a Boolean expression preceded with a |.

    Similarly to patterns, guards let us define a function on a case-by-case basis. The first guard that evaluates to True determines the return value.

    Example.

    The following function maps an integer to its absolute value:

    abs:: Integer -> Integer
abs x | x >= 0 = x
      | x < 0 = -x

    There are two guard in this definition:

    • | x >= 0
    • | x < 0

    Note.

    The function above is equivalent to

    abs:: Integer -> Integer
abs x | x >= 0 = x
      | True = -x

    Haskell provides the keyword otherwise as an alias for True.

    Example.

    The function above can be written

    abs:: Integer -> Integer
abs x | x >= 0 = x
      | otherwise = -x

    Exercise.

    Simplify the following function.

    dummyComp:: Integer -> Integer -> Bool
dummyComp x y | x <= y = True
              | x < y || x + 1 <= y = False
              | otherwise = True

    Solution.

    dummyComp:: Integer -> Integer -> Bool
dummyComp _ _ = True

    Syntax.

    A guard can be used together with a pattern.

    Example

    fib :: Int -> Int
fib 0 = 0                     -- pattern only
fib 1 = 1                     -- pattern only
fib n | n > 1 = fib (n-1) + fib (n-2)   -- pattern + guard

    Syntax.

    A pattern can also be used inside a guard, with a left arrow:

    | <pattern> <- <expression> = <result>

    The evaluation goes as follows:

    1. <expression> is evaluated first.
    2. Its value is matched against the pattern <pattern>.
    3. If the match succeeds, the guard is considered True and the right‑hand side (result) is selected.
    4. Otherwise control moves to the next next guard (if any).

    This can be helpful when <expression> is a call to another function.

    Example.

    -- this function maps a character a pair that consists of a number and a Boolean
analyzeChar :: Char -> (Int, Bool)
analyzeChar c = (unicodeCodePoint c, c
fib 0 = 0                     -- pattern only
fib 1 = 1                     -- pattern only
fib n | n > 1 = fib (n-1) + fib (n-2)   -- pattern + guard

    Conditional statement #

    Haskell also supports “if then else” expressions, similar top their counterparts in Java (and many programming languages).

    Example.

    The following function maps:

    • a (strictly) positive integer to 1,
    • 0 to 0 and
    • a negative integer to -1.
    signum:: Integer -> Integer
signum x = if x > 0 then 1 else
           if x == 0 then 0 else - 1

    Warning. As opposed to Java, the else case is mandatory.

    For instance, the following function definition is not syntactically valid.

    invalid:: Integer -> Integer
invalid _ = if True then 0

    When there are more than 2 cases, guards can be easier to read than nested conditional statements.

    Example.

    Compare the function signum above to the same function written with guards:

    signum:: Integer -> Integer
signum x | x > 0 = 1
         | x == 0 = 0
         | otherwise = -1
