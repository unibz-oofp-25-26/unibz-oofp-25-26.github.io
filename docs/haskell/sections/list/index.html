    List #

    Recursive form #

    Lists in Haskell are recursive.

    Definition. In Haskell:

    • [] is the empty list,
    • if xs is a list and x an element, then x:xs is a list.

    In addition, all elements in a list must have the same type. If this type is a, then the list has type [a].

    Example.

    The Haskell list 5 : (2 : []) stands for the mathematical list $5,2$. [Int] is a valid type for this list (as well as [Integer] or [Float]), because Int is a valid type for 5 and 2 .

    Warning. The empty list [] has type [a] for any type a.

    Examples.

    The following types are valid for the empty list (among other):

    • [Int] (list of bounded integers)
    • [Integer] (list of arbitrary integers)
    • [String] (list of strings)
    • [[String]] (list of lists of strings)
    • etc.

    The operator : (pronounced “cons”) associates to the right, so parentheses are often redundant.

    Examples.

    list in Haskell omitting parentheses
    $2$ 2 : [] 2 : []
    $5, 2$ 5 : (2 : []) 5 : 2 : []
    $3, 5, 2$ 3 : (5 : (2 : [])) 3 : 5 : 2 : []
    $\textit{b,d,a}$ 'b' : ('d' : ('a' : [])) 'b' : 'd' : 'a' : []

    Terminology. If x:xs is a list, then:

    • x is called its head,
    • xs is called its tail.

    The head is intuitively the first element of the list, and the tail is the rest of it.

    Note. If the list x : xs has type [a], then

    • x has type a, and
    • xs has type [a].

    Syntax. As syntactic sugar, a list can also be written in non-recursive form, within square brackets.

    Examples.

    Haskell list brackets notation
    2 : [] [2]
    5 : 2 : [] [5,2]
    3 : 5 : 2 : [] [3,5,2]
    'b' : 'd' : 'a' : [] ['b','d','a']

    Concatenation #

    Syntax. The operator ++ concatenates two lists.

    Examples.

    expression evaluation
    [2,1] ++ [3,2,5] [2,1,3,2,5]
    [2,1] ++ [] [2,1]
    [] ++ [3,2,5] [3,2,5]

    List patterns #

    • [] is a pattern. It matches the empty list.
    • If p and q are patterns, then (p:q) is a pattern. It matches a list whose head is matched by p, and whose tail is matched by q.
    • If p is a pattern, then [p] is a pattern, equivalent to (p:[]).
    • If p and q are patterns, then [p,q] is a pattern, equivalent to (p:q:[]).
    • etc.

    Example. The following function discards the first element of a list, unless the list has length $\le 1$.

    discardHead :: [Int] -> [Int]
discardHead [] = []
discardHead [x] = [x]
discardHead (_:xs) = xs

    Write a function startsWithTwins that maps a list of pairs of strings to:

    • True if its head contains twice the same string, and
    • False otherwise

    Possible solution

    startsWithTwins :: [(String,String)] -> Bool
startsWithTwins [] = False
startsWithTwins ((x,y) : _) = x == y

    List comprehension #

    We have seen [earlier] the (mathematical) syntax for set comprehensions.

    Example (reminder). If $X$ is a set of numbers, then the expression

    $\qquad\{x^2 \mid x \in X, x \ge 5\}$

    means “all $x^2$ such that $x \in X$ and $x \ge 5$”.

    This is the set that consists of all numbers greater or equal to $5$ in $X$, each squared.

    Haskell provides a similar syntax for lists (Python as well, since Python 2 in 2000).

    Example. In Haskell, if xs is a list of numbers, then we can write

    $\qquad$ [x^2 | x <- xs, x >= 5]

    This expression evaluates to the list of all x^2 such that x is in xs and x >= 5.

    Terminology. In the example above, the expression x <- xs is called a generator.

    A list comprehension may have several generators.

    Example. Here is a list comprehension with two generators:

    $\qquad$ [x + y | x <- xs, y <- ys]

    It evaluates to the list of all x + y such that x is in xs and y is in ys.

    Warning. For a list (as opposed to set) comprehension:

    • the order of the output list reflects the order of the initial list(s),
    • the output list may contain duplicates.

    Examples.

    • [x^2 | x <- [3,5,2]] evaluates to [9,25,4]
    • [x^2 | x <- [-3,3]] evaluates to [9,9]
    • [x^2 | x <- [3,5,2], x < 5] evaluates to [9,4]
    • [x + y | x <- [3,5,2], y <- [2,3]] evaluates to [5,6,7,8,4,5]

    Exercise

    Write a function firstsIfSecPos that maps a list of pairs of integers to a list that consists of the first element of each pair, if the second element is $\ge 0$.

    For instance firstsIfSecPos [(3,4), (5,-2), (-3,6), (1,3)] evaluates to [3, -3, 1]

    Solution

    firstsIfSecPos:: [(Int, Int)] -> [Int]
firstsIfSecPos ps = [x | (x,y) <- ps, y >= 0]
