    Abstract data type

    • Implementation
    • Abstraction

    Abstract data type #

    An abstract data type (ADT) is an interface to access and manipulate a collection of elements.

    Example. A queue is an ADT that mimics the behavior of a “first come first served” queue (e.g. a waiting line at a post office).

    It typically exposes three methods:

    • check whether the queue is empty,
    • take as input an element and adds it to the end of the queue,
    • remove the first element of the queue and return it.

    Implementation #

    An ADT may have several implementations.

    Example (continued). A queue may be implemented as an array or as a linked list (among others).

    Each implementation may have advantages and drawbacks (in terms of performance, memory footprint, parallelization opportunities, etc.) for different tasks (reading, writing, etc.).

    Terminology. A data structure is a concrete way to store a collection of elements (values, objects, etc.) in memory.

    Examples. Data structures include arrays, linked list, hash table, etc.

    Warning. A data structure is not an ADT.

    Instead, it can be used to implement one or several ADT(s).

    Abstraction #

    An ADT may correspond to a simple mathematical object (set, tuple, function, etc.). This is an intuitive, high-level way to manipulate a collection.

    Example. The ADT set is a convenient interface to manipulate collections with:

    • no duplicate, and
    • whose elements are in no specific order.

    Disclaimer. Due to limited time, we will only illustrate ADTs with Java (even though several of them have counterparts in Haskell).

    Note. Haskell lists may be viewed as an ADT.
