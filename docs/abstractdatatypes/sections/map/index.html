    Associative array #

    An associative array (or dictionary or map) simulates a function with finite domain.

    Equivalently, it can viewed as a finite set of key-value pairs, where keys are distinct.

    Example. The following associative array maps a city to its country:

    $\qquad \{ \textit{Verona} \mapsto \textit{Italy},\ \textit{Rome} \mapsto \textit{Italy},\ \textit{Innsbruck} \mapsto \textit{Austria} \}$

    The keys are $\textit{Verona}, \textit{Rome}$ and $\textit{Innsbruck}$.
    The values are $\textit{Italy}$ and $\textit{Austria}$.

    An associative array exposes at least the following methods:

    • “lookup” (or “get”) takes a key as input, and returns the value for this key (if any).
    • “insert” (or “put”) inserts a pair (key, value). If an entry for this key was already present, then it overwrites its value.
    • “remove” (or “delete”) deletes the entry for a given key (if any).

    Note. Associative arrays are pervasive in programming. Many data structures have been designed over the years whose main purpose is to implement associative arrays efficiently (notably hash tables and a variety of search trees).

    in Java #

    Java has an interface Map with 19 native implementations (meaning that 19 different classes implement this interface).

    The most frequently used implementation is HashMap.

    Note. In contrast to List and Set, the interface Map does not extend the interface Collection.

    Map is a generic type with two parameters: one for the type of its keys, and one for the type of its values.

    Example. A map from integers to strings has type Map<Integer, String>.

    Syntax #

    Consider the classes City used in the section on sets. Here are code snippets for a few operations specified in the interface Map.

    Create a Map and populate it:

    City milan = new City("Milan", 20100);
City florence = new City("Florence", 50100);

// create an empty map whose keys are zipCodes,
// and whose values are cities
Map<Integer, City> zipCodeToCity = new HashMap<>();

// add the entry { 20100 ↦ milan } to the map
zipCodeToCity.put(milan.zipCode, milan);

// add the entry { 50100 ↦ florence } to the map
zipCodeToCity.put(florence.zipCode, florence);

// creates a map identical to the previous one,
// but which cannot be modified
Map<Integer, City> anotherMap = Map.of( milan.zipCode, milan,
                                        florence.zipCode, florence );

    Retrieve the value associated to a given key:

    // contains (a reference to) Milan
City aCity = zipCodeToCity.get(20100);

// has value null
City anotherCity = zipCodeToCity.get(88888);

    Check whether a map contains an entry for a given key:

    // outputs true
System.out.println(zipCodeToCity.containsKey(20100));

// outputs false
System.out.println(zipCodeToCity.containsKey(88888));

    Retrieve the number of entries in a map:

    // outputs 2
System.out.println(zipCodeToCity.size());

    Overwrite an entry in a map:

    City bologna = new City("Bologna", 40100);

// replace { 20100 ↦ milan } with { 20100 ↦ bologna }
zipCodeToCity.put(20100, bologna);

// outputs 2
System.out.println(zipCodeToCity.size());

// add { 40100 ↦ bologna } to the map
zipCodeToCity.put(bologna.zipCode, bologna);

// outputs 3
System.out.println(zipCodeToCity.size());

// replace { 20100 ↦ bologna } with { 20100 ↦ milan }
zipCodeToCity.put(milan.zipCode, milan);

    Retrieve the set of keys of a map (this is an instance of Set<E>, where E is the type of the keys)

    Set<Integer> keys = zipCodeToCity.keySet();

// outputs true
System.out.println(keys.contains(20100));

    Retrieve a collection with all values in the map, in no specific order (may contain duplicates):

    Collection<City> values = zipCodeToCity.values();

// outputs true
System.out.println(values.contains(milan));

// outputs 3
System.out.println(values.size());

// add { 99999 ↦ milan } to the map
zipCodeToCity.put(99999, milan);

// outputs 4
System.out.println(zipCodeToCity.values().size());

    Wrap the map into a set of entries (a.k.a. “key-value” pairs):

    // create a "wrapper" `Set` around the map (no data is duplicated)
// each element of this set has type `Entry<Integer,City>`
Set<Entry<Integer,City>> entries = zipCodeToCity.entrySet();

// iterate over entries
for (Entry<Integer, City> entry : entries){
    // holds the entry's key
    Integer zipCode = entry.getKey();
    // holds the entry's value
    City city = entry.getValue();
}

    Again, we refer to the Javadoc for an exhaustive documentation.

    Comparing keys #

    The identity of two keys is determined by the method equals, similarly to what we have seen for sets.

    Example. Consider the classes City and SmartCity used in the section on sets, where SmartCity overrides equals (and hashCode), whereas City does not:

    City trento = new City("Trento", 38100);
City trentoAgain = new City("Trento", 38100);
Country italy = new Country("Italy");

Map<City, Country> cityToCountry = new HashMap();
cityToCountry.put(trento, italy);
cityToCountry.put(trentoAgain, italy);

// outputs 2
System.out.println(cityToCountry.size());

SmartCity smartTrento = new SmartCity("Trento", 38100);
SmartCity smartTrentoAgain = new SmartCity("Trento", 38100);

Map<SmartCity, Country> smartCityToCountry = new HashMap();
smartCityToCountry.put(smartTrento, italy);
smartCityToCountry.put(smartTrentoAgain, italy);

// outputs 1
System.out.println(smartCityToCountry.size());

    Usage #

    Implement the method

    int countRepeatedChars(char[] chars)

    such that countRepeatedChars(chars) is the number of characters that appear at least $3$ times in chars.

    For instance, if chars is the array ['a','e','b','c','b','a','d','b'], then countRepeatedChars(chars) should evaluate to 1.

    Constraint. Your implementation should not sort the array.

    Consider the Java record

    record City(String name, String country) { }

    Implement the method

    void printNumberOfCitiesByCountry(City[] cities)

    such that printNumberOfCitiesByCountry(cities) prints the number of cities for each country in the array cities, in any order.
    For instance,

    City verona = new City("Verona", "Italy");
City rome = new City("Rome", "Italy");
City innsbruck = new City("Innsbruck", "Austria");

printNumberOfCitiesByCountry(new City[]{verona, rome, innsbruck, rome});

    should print

    "Italy": 2
"Austria": 1

    or

    "Austria": 1
"Italy": 2
