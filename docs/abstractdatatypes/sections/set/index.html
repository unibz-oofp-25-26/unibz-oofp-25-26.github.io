    Set #

    The abstract datatype set simulates a (usually finite) set.

    A set may expose the following methods:

    • “add”: adds an element to the set if it was not present already (and returns true iff this was the case),
    • “contains”: checks whether an element belongs to the set,
    • “size”: returns the cardinality of the set,
    • etc.

    Warning. By default, there is no guarantee on the order of elements in a set.

    in Java #

    Java has an interface Set with 8 native implementations (meaning that 8 different classes implement this interface). The most frequently used Implementations are HashSet and TreeSet.

    The interface Set extends the interface Collection.

    Syntax #

    Here are code snippets for a few simple operations specified in the interface Set.

    Create a Set and populate it:

    City milan = new City("Milan", 20100);
City florence = new City("Florence", 50100);

// create an empty set of cities
// alternatively, we could use another implementation, like `TreeSet`
Set<City> mySet = new HashSet<>();

// add Milan to the set
mySet.add(milan);
// add Florence to the set
mySet.add(florence);
// try to add Milan again, which has no effect
mySet.add(milan);

// create a set identical to the previous one,
// but which cannot be modified
Set<City> myOtherSet = Set.of(milan, florence);

    Remove an element from a set:

    mySet.remove(milan);

    Check whether a set contains a certain element:

    // outputs false
System.out.println(mySet.contains(milan));

// outputs true
System.out.println(mySet.contains(florence));

    Get the cardinality of a set:

    // outputs 1
System.out.println(mySet.size());

    Compute the intersection of two sets:

    mySet.add(new City("Bologna", 40100));
mySet.retainAll(myOtherSet);

// outputs 1
System.out.println(mySet.size());

    Add a Collection to a set:

    mySet.addAll(myOtherSet);

// outputs 2
System.out.println(mySet.size());

    For more operations, consult the Javadoc of the interface Set.

    Duplicates #

    By definition, a set cannot contain identical elements.

    Question. What does “identical” mean for two objects?

    In Java, the method equals (implicitly) determines whether two elements added to a Set should be considered identical.

    Warning. Recall that the method equals should be overridden together with the method hashCode. This is necessary for a HashSet to behave as expected (more on this later).

    Example. Consider a naive implementation of the class City, which does not override equals (or hashCode):

    public class City {

    String name;
    int zipcode;

    public City(String name, int zipCode){
        this.name = name;
        this.zipCode =zipCode;
    }
}

    Recall that every class inherits a default implementation of equals (and hashCode), from the class Object. According to this default implementation, equals behaves like ==.

    City trento = new City("Trento", 38100);
City trentoAgain = new City("Trento", 38100);

Set<City> cities = new HashSet<>();
cities.add(trento);
cities.add(trentoAgain);

// outputs 2
System.out.println(cities.size());

    Now consider a class SmartCity identical to City, but that overrides equals (andhashCode) in the standard way:

    public class SmartCity {
  String name;
  int zipCode;

  public SmartCity(String name, int zipCode) {
    this.name = name;
    this.zipCode = zipCode;
  }

  @Override
  public boolean equals(Object o) {
    if (this == o) return true;
    if (o == null || getClass() != o.getClass()) return false;
    SmartCity smartCity = (SmartCity) o;
    return zipCode == smartCity.zipCode && name.equals(smartCity.name);
  }

  @Override
  public int hashCode() {
    return Objects.hash(zipCode, name);
  }
}

    A Java Set cannot contain two instances of SmartCity that have the same name and zip code:

    SmartCity smartTrento = new SmartCity("Trento", 38100);
SmartCity smartTrentoAgain = new SmartCity("Trento", 38100);

Set<SmartCity> smartCities = new HashSet<>();
smartCities.add(smartTrento);
smartCities.add(smartTrentoAgain);

// outputs 1
System.out.println(smartCities.size());

// outputs false
System.out.println(smartCities.add(smartTrentoAgain));

    Reminder. Many native Java classes already override equals (and hashCode), in the expected way.
    These include:

    • String,
    • boxed types (like Integer),
    • most implementations of the Collection interface (including HashSet).

    Note. Some implementations of List also provide methods that perform “set-like” operations, like containsAll, removeAll, retainAll, etc. Again, we refer to the Javadoc for an exhaustive documentation.

    Usage #

    The ADT “set” can be useful in a variety of contexts.

    Example. We can use a set to remove duplicates in an array (if the order of elements in the array is irrelevant):

    Integer[] myArray = new Integer[]{1,2,3,1,2};
// the constructor of `HashSet` cannot take an array as input, so we wrap the array into a list
Set<Integer> set = new HashSet<>(Arrays.asList(myArray));

// outputs 3
System.out.println(set.size());

    Example. We can use sets to check whether two lists contain the same (distinct) elements:

    City mantua = new City("Mantua", 46100);
City bergamo = new City("Bergamo", 24100);
List<City> list1 = List.of(mantua, bergamo);
List<City> list2 = List.of(bergamo, mantua, bergamo);

Set<City> set1 = new HashSet<>(list1);
Set<City> set2 = new HashSet<>(list2);

// outputs true
System.out.println(set1.equals(set2));

    Consider a version of our game where each NPC stores a list of encountered NPCs (possibly with duplicates), in chronological order.

    Let us assume that none of NPC, Unicorn or Butterfly overrides equals or hashCode (which is sensible in this example).

    Implement the method

    int countUniFriends(NPC npc)

    such that countUniFriends(npc) is the number of distinct unicorns that c has met.

    int countUniFriends(NPC npc) {

    Set<NPC> unicorns = new HashSet<>();

    for (NPC encounter : npc.encounters) {
        if (encounter instanceof Unicorn) {
            unicorns.add(encounter);
        }
    }
    return unicorns.size();
}
