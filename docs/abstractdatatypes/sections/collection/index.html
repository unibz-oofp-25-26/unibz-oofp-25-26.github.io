    ADT as a generic interface #

    An ADT in Java is usually a generic type, meaning that it accepts one or several types as parameter(s).

    Example. A list of integers has type List<Integer>.

    Syntax. The type List<E> is a generic type, with type variable E

    Note. List<E> in Java is analogous to [a] in Haskell.

    Constraint. In Java, a primitive type (like int or bool) cannot be a type parameter, whereas a boxed type (like Integer or Boolean) can.

    The interface Collection #

    Many Java ADTs (and data structures) extend or implement the interface called Collection (the only exception in this chapter is Map).

    Syntax. The interface Collection specifies the following instance methods (among others):

    • int size()
    • boolean add(E e)
    • boolean contains(Object o)
    • boolean isEmpty()
    • boolean remove(E e)
    • etc.

    where E is the type of the elements in the collection.

    Warning. Some of these methods are called “optional operations”, meaning that they may not be available for all implementations of Collection.

    Iteration #

    Like an array, a collection can be iterated over, for instance with a “foreach”-like loop:

    for (E element: myCollection){
      ...
}

    Example.

    // create a list
List<Integer> myList = List.of(1,2,3);
// iterate over it
for (Integer i: myList){
    System.out.println(i);
}

    Another way to iterate over a collection consists in using an iterator.

    Warning. Some collections (e.g. some implementations of the subinterface Set) do not offer any guarantee on the order of their elements. So iterating over such a collection is not fully predictable.

    External libraries #

    Some external libraries provide additional implementations of Java’s ADTs.

    Example. Java has 19 native implementations of the ADT Map, like HashMap or TreeMap.

    Guava provides additional ones, like ImmutableMap or HashBiMap.
