    Subtype #

    In object-oriented languages (like Java), subtyping generalizes the notion of subclass.

    Intuitively, $A$ is a subtype of $B$ if “every $A$ is a $B$”.

    Examples. In Java, $A$ is a subtype of $B$ if:

    • $A$ and $B$ are classes and $A$ extends $B$ (directly or transitively), or
    • $A$ and $B$ are interfaces and $A$ extends $B$ (directly or transitively), or
    • $A$ is a class, $B$ is an interface and $A$ implements $B$ (directly or transitively), or
    • etc.

    The subtype relation #

    Notation. We will use $A \sqsubseteq B$ to indicate that $A$ is a subtype of $B$.

    Examples. In Java,

    • Integer $\sqsubseteq$ Number
    • String $\sqsubseteq$ Object

    Observation. $\sqsubseteq$ is a binary relation. This relation is both:

    • reflexive: $A \sqsubseteq A$ for every type $A$

    • transitive: $A \sqsubseteq B$ and $B \sqsubseteq C$ imply $A \sqsubseteq C$

    Observation. If an object $o$ has type $A$ and $A \sqsubseteq B$, then $o$ also has type $B$.

    Examples. In Java:

    • if $o$ has type Integer, then $o$ also has type Number
    • if $o$ has type String, then $o$ also has type Object

    Method subtyping #

    Question. How can we generalize $\sqsubseteq$ to method types?

    To answer this question, we will first use mathematical functions as an analogy.

    Domain, codomain and set inclusion #

    Reminder. A function $f\colon X \to Y$ maps every element of $X$ to some element of $Y$.

    Glossary (reminder).

    • $\mathbb{N}$ : natural numbers (a.k.a. positive integers, including 0)
    • $\mathbb{Z}$ : integers
    • $\mathbb{Q}$ : rational numbers

    Observe that $\mathbb{N} \subseteq \mathbb{Z} \subseteq \mathbb{Q}$.

    Example. Let $\mathsf{ItalianCities}$ denote the set of all Italian cities.

    Consider the function $\mathit{pop}$ that maps an Italian city to its number of inhabitants.
    We can write

    $\qquad \mathit{pop}\,\colon \mathsf{ItalianCities} \to \mathbb{N}$

    because $\mathit{pop}$ maps every Italian city to some natural number.

    We can also write

    $\qquad \mathit{pop}\,\colon \mathsf{ItalianCities} \to \mathbb{Z}$

    because $\mathit{pop}$ maps every Italian city to an integer (since natural numbers are integers).

    For the same reason, we can also write

    $\qquad \mathit{pop}\,\colon \mathsf{ItalianCities} \to \mathbb{Q}$

    However, we cannot write

    $\qquad \mathit{pop}\,\colon \mathsf{Cities} \to \mathbb{N}$

    because $\mathit{pop}$ does not map every city (only Italian ones).

    Method type #

    We can lift these operations to method types, using $\sqsubseteq$ instead of $\subseteq$.

    Consider a programming language with a types for $\mathbb{N}$, $\mathbb{Z}$ and $\mathbb{Q}$ (this can be simulated with the java.math library). In this language, we naturally have $\mathbb{N} \sqsubseteq \mathbb{Z} \sqsubseteq \mathbb{Q}$.

    Now consider a method $f$ with type $\mathbb{Q} \to \mathbb{Q}$ that maps a number $x$ to $x^2$.

    Which of the following types are also valid for $f$ ?

    1. $\qquad \mathbb{Q} \to \mathbb{Z}$
    2. $\qquad \mathbb{Q} \to \mathbb{N}$
    3. $\qquad \mathbb{Z} \to \mathbb{Q}$
    4. $\qquad \mathbb{Z} \to \mathbb{Z}$
    5. $\qquad \mathbb{Z} \to \mathbb{N}$
    6. $\qquad \mathbb{N} \to \mathbb{Q}$
    7. $\qquad \mathbb{N} \to \mathbb{Z}$
    8. $\qquad \mathbb{N} \to \mathbb{N}$

    In our language, which of the following are correct?

    A method with type $\mathbb{Z} \to \mathbb{Z}$ also has type:

    1. $\qquad \mathbb{N} \to \mathbb{N}$
    2. $\qquad \mathbb{N} \to \mathbb{Z}$
    3. $\qquad \mathbb{N} \to \mathbb{Q}$
    4. $\qquad \mathbb{Z} \to \mathbb{N}$
    5. $\qquad \mathbb{Z} \to \mathbb{Q}$
    6. $\qquad \mathbb{Q} \to \mathbb{N}$
    7. $\qquad \mathbb{Q} \to \mathbb{Z}$
    8. $\qquad \mathbb{Q} \to \mathbb{Q}$

    Generalization #

    We can now generalize these observations:

    Property.

    $\qquad$ $X_1 \sqsubseteq X_2$ and $Y_1 \sqsubseteq Y_2$

    imply

    $\qquad X_2 \to Y_1 \sqsubseteq X_1 \to Y_2$

    Example. If Butterfly $\sqsubseteq$ NPC, then a Java method with type NPC $\to$ Integer also has type:

    • Butterfly $\to$ Integer
    • NPC $\to$ Number
    • Butterfly $\to$ Number

    or in other words:

    • NPC $\to$ Integer $\sqsubseteq$ Butterfly $\to$ Integer
    • NPC $\to$ Integer $\sqsubseteq$ NPC $\to$ Number
    • NPC $\to$ Integer $\sqsubseteq$ Butterfly $\to$ Number
