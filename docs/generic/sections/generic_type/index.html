    Generic type #

    In Haskell #

    Syntax #

    We have already seen how to define a product type or a sum type.

    Example. As we have seen earlier, the type of lists of integers can be defined as follows:

    data IntList = Nil | Cons Int IntList

    Similarly, the type of lists of Booleans can be defined as

    data BoolList = Nil | Cons Bool BoolList

    To make this more generic, we can use a type variable instead of the concrete type (Int or Bool here).

    Syntax. A generic type in Haskell is declared as

    data <typeName> <type variable 1> ... <type variable n> = <definition>

    where <definition> is a type definition that may use the type variables declared on the left-hand side.

    Example (continued). The parameterized type “list of a’s” (where a is a type variable) can be defined analogously to the two types above, as

    data List a = Nil | Cons a (List a)

    This type is equivalent to the native Haskell type [a].

    in Java #

    Syntax. A generic type in Java is immediately followed by type variable declarations, between < >.

    Examples.

    • Set<E> is a generic type,

    • Set<T> is equivalent to Set<E>,

    • Function<T,R> is another generic type.

    Scope #

    Syntax. The variables of a generic type are visible in the whole class (or interface, record, etc.) that defines the type.

    Example. We can create a generic wrapper class Box (similar to Optional), as follows:

    public class Box<T> {

    T boxedValue;

    public Box(T boxedValue){
        this.boxedValue = boxedValue;
    }

    T get(){
        if(boxedValue != null) {
            return boxedValue;
        }
        ...
    }
    ...
}

    Restriction. The value of a type variable may vary from one instance of a generic type to the other. Therefore a type variable associated to a class (resp. interface) cannot be used:

    • in a static method of this class (resp. interface), or
    • as the type of a static attribute of this class (resp. interface).

    Example. The following class does not compile:

    public class Box<T> {

    static T myAttribute;
    ...
}

    Instantiation #

    Syntax. When can create an instance of a generic class (resp. record) by providing parameter types when calling its constructor.

    Example (continued).

    Box<String> myBoxedString = new Box<String>("Hi there");

    Syntactic sugar. If the parameters are redundant, then we can omit them and write <> instead.

    Example (continued). In the example above, the second occurrence of String is redundant. It can be safely be omitted:

    Box<String> myBoxedString = new Box<>("Hi there");

    Inheritance #

    Constraint. A subclass must declare the type variables that it uses to refer to its superclass.

    Example. The following program does not compile, because the subclass uses T without declaring it.

    public class Superclass<T> {
      ...
}

public class Subclass extends Superclass<T> {
    ...
}

    In contrast, the following are all valid:

    public class Superclass<T> {
      ...
}

public class Subclass<T> extends Superclass<T> {
    ...
}

    $\qquad$

    public class Superclass<T> {
      ...
}

public class Subclass<T extends NPC> extends Superclass<T> {
    ...
}

    $\qquad$

    public class Superclass<T,V> {
      ...
}

public class Subclass<T> extends Superclass<T,T> {
    ...
}

    $\qquad$

    Syntax. A generic subtype can also declare additional type variables.

    Example.

    public class Superclass<T> {
      ...
}

public class Subclass<T,V> extends Superclass<T> {
    ...
}

    Generalization. These rules also apply to:

    • an interface that extends an interface,
    • a class that implements an interface,
    • etc.
