    Generic type #

    In Haskell #

    Syntax #

    We have seen how Haskell uses constructors to define composite types.

    Example (reminder). As we have seen earlier, the type of lists of integers can be defined as follows:

    data IntList = Nil | Cons Int (IntList)

    Similarly, the type of lists of Booleans can be defined as

    data BoolList = Nil | Cons Bool (BoolList)

    If we want to make this more generic, we can use a type variable instead of the concrete type (Int or Bool here).

    Syntax. A parameterized type declaration in Haskell has the form

    data <typeName> <first type variable> ... <last typeVariable> = <definition>

    where <definition> is a type definition that may use the type variables declared on the left-hand side.

    Example (continued). The parameterized type “list of a’s” (where a is a type variable) can be defined analogously to the two types above, as

    data List a = Nil | Cons a (List a)

    This type is equivalent to the native Haskell type [a] that we have used in the previous section.

    Example. Recall the type IntBT of binary trees with integer values that we used earlier:

    data IntBT = Empty | Node Int IntBT IntBT

    We can also declare a parameterized version BT aof this type, where nodes carry a value of type a:

    data BT a = Empty | Node a (BT a) (BT a)

    Generic functions #

    Arbitrary type parameter #

    As we have seen already, the parameterized type [a] lets us write functions for arbitrary lists (regardless of the specific type of elements in a list).

    Example (reminder)

    length :: [a] -> Int
length [] = 0
length (_ : xs) = 1 + length xs

    Similarly, we can modify some of the functions that we wrote for a tree of integers, so that they work for arbitrary tree.

    Example. Recall the function height :: IntBT -> Int that we have seen in the section on binary recursion.

    data IntBT = Empty | Node Int IntBT IntBT

height :: IntBT -> Int
height Empty = 0
height Node i t1 t2 = 1 + max (height t1) (height t2)

    The same implementation would work for a tree of strings, a tree of Boolean values, etc.

    So we can use our parameterized type BT a to write a generic version of our function:

    data BT a = Empty | Node a (BT a) (BT a)

height :: BT _ -> Int
-- Same implementation as the function `height :: IntBT -> Int above`
<...>

    Observe that the implementation is identical (only the type of the function is affected).

    Note. In this example, we could equivalently have used the declaration

    height :: BT a -> Int

    We used the wildcard type _ instead of a for clarity, because the value of a in this example is irrelevant.

    Bounded type parameter #

    Example (reminder). Recall the function contains :: Int -> IntBT -> Bool that we have seen in the section

    data IntBT = Empty | Node Int IntBT IntBT

contains :: Int -> IntBT -> Bool
contains Empty = False
contains i Node j t1 t2 = i == j || contains i t1 || contains i t2

    As seen int the previous section, we cannot replace Int with any type here, because the function makes use of equality. So we can only generify this function to types that support equality, as follows:

    data BT a = Empty | Node a (BT a) (BT a)

contains :: Eq a => a -> BT a -> Bool

    Again, the implementation of the function is unchanged (only its type is affected).
