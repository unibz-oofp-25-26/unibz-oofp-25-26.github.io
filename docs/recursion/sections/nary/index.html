    N-ary recursion #

    Terminology. A recursive method that is not linear recursive uses n-ary recursion.

    Specific cases include:

    • binary recursion if a method performs up to two recursive calls each time it is executed,
    • ternary recursion if it performs up to three recursive calls each time it is executed,
    • etc.

    In an imperative language (like Java), a linear recursive algorithm can easily be converted into an iterative one (i.e. an algorithm that uses loops). This is often not the case with n-ary recursion.

    Due to limited time, we will focus on Java (rather than Haskell) in this course to illustrate n-ary recursion.

    Example: binary trees #

    Definition. An ordered tree is a tree where the children of each node are totally ordered.

    Definition. A binary tree is an ordered tree where each node has exactly two (possibly null) children.

    Example. This is a binary tree:

    Let us represent a node in a binary tree as an instance of the following Java class:

    Implement a method

    int sum(Node root)

    such that sum(root) is the sum of all values in the binary tree with root root.

    For instance, for the following tree, the method should return 13:

    Hint. Recall the methodology that we have seen earlier to solve a problem recursively.

    int sum(Node root){

    // base case
    if (root == null){
        return 0;
    }
    // inductive case
    return root.value + sum(root.leftChild) + sum(root.rightChild);
}

    Let us represent a binary tree as above, but where nodes have no label:

    Implement a method

    int height(Node root)

    such that height(root) is the number of nodes in the longest branch of the binary tree with root root.

    For instance, for the following tree, the method should return 4:

    Syntax. In Java, the static method

    int max(int x, int y)

    of the class Math returns the maximum of x and y.

    int height(Node root){
    //base case
    if (root == null){
        return 0;
    }
    // inductive case
    return 1 + max(
                height(root.leftChild),
                height(root.rightChild)
    );
}

    Definition. In the above exercises, the three node processing orders are respectively called:

    • pre-order traversal,
    • in-order traversal and
    • post-order traversal.

    These three approaches can be adapted to solve a variety of problems.

    Hint. In-order traversal processes nodes in left-to-right order (when projected to a line).

    Let us consider once again a binary tree where nodes are labeled with a character.

    Implement a method

    Node leftMost(Node root, char label)

    such that leftMost(root, label) is the leftmost node labeled with label in the tree with root root (or null if there is no such node).

    We can adapt the in-order traversal algorithm (above), as follows:

    Node leftMost(Node root, char label){
    // Base case.
    if(root == null){
        return null;
    }

    // Inductive case.
    // Explore the left subtree first.
    Node candidate = leftMost(root.leftChild, label);
    // If the call succeeded, then we found the leftmost occurrence in the tree.
    if(candidate != null){
        return candidate;
    }
    // Otherwise the next candidate (from left to right) is the current node.
    if (root.label == label) {
        return root;
    }
    // If both attempts failed, then explore the right subtree.
    return leftMost(root.rightChild, label);
}

    Divide-and-conquer #

    Divide-and-conquer is a variation of the methodology seen earlier to solve a problem recursively. In the divide-and-conquer approach, the smaller inputs $I_1$, .., $I_k$ usually have (nearly) the same size.

    Divide-and-conquer has been instrumental in discovering efficient solutions to pervasive computational problems. Well-known examples include:

    • the Karatsuba algorithm for multiplying two numbers,
    • the Strassen algorithm for matrix product,
    • Mergesort for (stable) sorting.

    Example: Mergesort #

    Mergesort is one of (many) algorithms commonly used to sort an array, i.e. to solve the following problem:

    Input: an array $A$

    Output: a sorted array with the same elements as $A$

    Observation. As we saw already, the array can be of any type (int, String, City, NPC, etc.), and the sorting criterion can be any total preorder over this type.

    The Mergesort algorithm may be summarized as follows:

    • (Base case). If $A$ has size $1$, then return it ($A$ is already sorted).
    • (Inductive case). Otherwise:
      • (divide): partition $A$ into its left half $A_1$ and its right half $A_2$,
      • (solve): sort $A_1$ and sort $A_2$ (recursively),
      • (combine): merge (the sorted versions of) $A_1$ and $A_2$.

    Here is a naive implementation, in pseudocode:

    Warning. This implementation is suboptimal (some arrays can be reused). We only used it to keep the explanation simple.

    // Returns a sorted array that contains the same elements as A.
T[] mergesort(T[] A) {

    // Base case: the input array has size 1, it is already sorted.
    if(A.length == 1){
        return A
    }

    // Inductive case (the input array has size > 1)

    // Compute the middle index (rounded down)
    int middleIndex = (A.length - 1) / 2

    // Copy the left half of A into a new array
    T[] leftHalf = copy(A[0 .. middleIndex])
    // Sort it
    leftHalf = mergesort(leftHalf)

    // Copy the right half of A into a new array
    T[] rightHalf =  copy(A[middleIndex + 1 .. A.length - 1])
    // Sort it
    rightHalf = mergesort(rightHalf)

    // merge the two sorted halves
    return merge(leftHalf, rightHalf)
}

    where the auxiliary method merge is analogous to the function mergeSorted that we have seen in the first assignment.

    In action. You can find visual illustrations of the execution of merge sort online. For instance this animation, or this (more accurate) one.

    Observation. Merge sort is a stable sorting algorithm.

    Tree and graph traversal #

    Earlier in this chapter, we used different methods to traverse a binary tree, where nodes were instances of the following class:

    One of these methods was the so-called “pre-order traversal”:

    void traverse(Node root){
    // inductive case only (do nothing in the base case)
    if (root != null){
        System.out.print(root.label);
        traverse(root.leftChild);
        traverse(root.rightChild);
    }
}

    This method naturally generalizes to trees where nodes may have more than two children. Nodes in such a tree can be represented as instances of the following class:

    And the method becomes:

    void traverse(Node root){
    if (root != null){
        System.out.print(root.label);
        for(Node child : root.children {
            traverse(child);
        }
    }
}

    This approach can in turn be adapted to traverse a graph. Precisely, to explore all nodes reachable (directly or transitively) from a given source node in the graph.

    Warning. If applied naively to a graph, the algorithm above:

    • may not terminate (if the graph contains a loop),
    • may process some nodes multiple times (even if the graph is acyclic).

    In order to avoid this, a technique consists in labelling nodes in a graph with an additional boolean attribute visited, which indicates whether a node has already been visited during a traversal:

    And the method becomes:

    void traverse(Node root){
    if (root != null && !root.visited){
        root.visited = true;
        System.out.print(root.label);
        for(Node child : root.children {
            traverse(child);
        }
    }
}

    Terminology. This approach is often called depth-first exploration of a graph.
