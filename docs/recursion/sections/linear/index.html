    Linear recursion #

    In this section, we will mostly use Haskell code as illustrations.

    Execution #

    Example.

    The following function sums the elements of a list of integers

    sum :: [Int] -> Int
sum [] = 0
sum (x : xs) = x + sum xs

    Let us clarify how an expression like sum [1,2,3] is evaluated.

    Fist, recall that [1,2,3] is syntactic sugar for 1 : (2 : (3 : [])).

    Therefore when the pattern (x : xs) (line XXX) is matched against [1,2,3]:

    • x matches 1, and
    • xs matches 2 : (3 : []) (or equivalently, [2,3])

    So the evaluation of sum [1,2,3] requires evaluating sum [2,3], recursively.

    Besides, the + operation is performed after termination of the recursive call.

    So the following sequence of events takes place:

    1. the evaluation of sum [1,2,3] starts
    2. $\quad$ the evaluation of sum [2,3] starts
    3. $\quad\quad$ the evaluation of sum [3] starts
    4. $\quad\quad\quad$ the evaluation of sum [] starts
    5. $\quad\quad\quad$ the evaluation of sum [] ends and returns 0
    6. $\quad\quad$ the evaluation of sum [3] ends and returns 3 + sum [] (i.e. 3)
    7. $\quad$ the evaluation of sum [2,3] ends and returns 2 + sum [3] (i.e. 5)
    8. the evaluation of sum [1,2,3] end ands returns 1 + sum [2,3] (i.e. 6)

    After Step 4, the call stack is:

    sum[]
sum[3]
sum[2,3]
sum[1,2,3]

    Writing a recursive function #

    Understanding the execution of a recursive program (as we did above) is mostly useful for debugging.

    In order to write a recursive program/function, one can very often use a simpler, high-level methodology:

    Methodology. To solve a problem recursively:

    • (Base case). If the input $I$ is minimal (e.g. of size 0 or size 1), then solve the problem directly.
    • (Inductive case). Otherwise:
      1. Decompose $I$ into one or several smaller input(s) $I_1$, .., $I_k$.
      2. Solve the problem for $I_1$, .., $I_k$ by performing recursive calls.
      3. Use the results(s) of the recursive calls to solve the problem for $I$.

    In other words, in the inductive case, solve the problem for $I$ under the assumption that it can be solved for $I_1$, .., $I_k$.

    Example. Let us reproduce the function above.

    sum :: [Int] -> Int
-- base case (the input list has size 0)
sum [] = 0
-- inductive case (the input list has size n + 1)
sum (x : xs) = x + sum xs

    In this example, the input $I$ is a list.

    • In the base case ($I$ is the empty list), we solve the problem directly by returning 0
    • In the inductive case, $I$ is of the form $a \colon I’$, where $a$ is an integer and $I’$ is a list. In this case, the sub-list $I’$ is our (unique) smaller input. We:
      • solve the problem for $I’$ by calling the function sum recursively, and
      • use the result to solve the problem for the full list $I$, assuming that the recursive call is correct.

    Hint. For pure functions, we can always assume that a recursive call is correct, as long as its input is smaller than the current input $I$.

    Correctness of the whole algorithm holds by induction on the size of the input.

    Example.

    In the example above:

    • If the input list has size $0$ (i.e. if it is empty), then the function is trivially correct.
    • If the input list has size $1$, then the recursive call (Line XXX) is correct (since it takes as input a list of size $0$), so the function is correct,
    • If the input list has size $2$, then the recursive call (Line XXX) is correct (since it takes as input a list of size $1$), so the function is correct,
    • …
    • If the input list has size $n+1$, then the recursive call (Line XXX) is correct (since it takes as input a list of size $n$), so the function is correct.

    Hint. When the input $I$ is a list, the tail of $I$ is often a good candidate for a recursive call.

posTailLengthRec :: [Bool] -> (Int, Bool)
posTailLengthRec [] = 0
posTailLengthRec (False : xs) = (False, second (posTailLengthTRec xs))
posTailLengthRec (True : xs)
          | (True, v)  <- posTailLengthTrec xs = (True, v + 1)
          | (False, v) <- posTailLengthTrec xs = (False, v)
