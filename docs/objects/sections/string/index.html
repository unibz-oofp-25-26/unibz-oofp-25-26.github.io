    String #

    Many languages (such as Java, C#, Python, Ruby, JavaScript, Go, etc.) use an optimization technique called interning in order to manage strings (or sometimes integers, etc.) in memory.

    Interning consists in storing a single copy of each distinct string created during the execution of a program. Interned strings are stored in a shared pool, and each of them is immutable (meaning that it cannot be modified).

    Benefits include faster string comparison, lower memory footprint, etc.

    In languages where strings are treated as (pseudo) objects, like Java, this can make behavior of some operations hard to predict.

    in Java #

    In Java, a string is an object.

    Comparing strings #

    As for regular objects, the == operator compares the references of two Java strings:

    Example.

    String myString = new String("foo");
String sameString = new String("foo");

// Outputs false, because the two variables hold different references.
System.out.println(myString == sameString);

    Explicit interning #

    Syntax. A string can be explicitly interned (i.e. added to the shared pool) with the (instance) method intern.
    This method:

    • tries to add the string to the pool, and then
    • returns a reference to the (only) version of the string contained in the pool.

    Example. Let us assume that no string has been created before the following is executed:

    // create a string "foo"
String myString = new String("foo")

// add it to the pool
myString = myString.intern();

// create another string "foo"
String sameString = new String("foo");

// outputs false
System.out.println(myString == sameString);

// try to add "foo" to the pool once again;
// since the pool already contains a version of "foo", returns a reference to it
sameString = sameString.intern();

// outputs true, because the two variables now hold the same reference
System.out.println(myString == sameString);

    Implicit interning #

    Warning. A string that is initialized without constructor (i.e. directly with " ") is interned.

    Example.

    String myString = "foo";
String sameString = "foo";

// outputs true, because of (implicit) interning
System.out.println(myString == sameString);

    Immutability #

    In order to support interning, strings in Java are immutable.

    Warning. As a consequence, the instance methods of the class String do not modify the current string, contrary to what their name may suggest (substring, replace, etc.).

    Instead, these methods (may) return (a reference to) a different object.

    Example.

    String foo = new String("foo");
String copy = foo;

// outputs true, because the two variables hold the same object reference
System.out.println(copy == foo);

foo = foo.replace('o','O');

// outputs false, because the method 'replace' returns a reference to another object
System.out.println(copy == foo);

    Equality #

    The class String overrides the method equals so that it implements string comparison in the expected way.

    Example.

    String myString = new String("foo");
String sameString = new String("foo");

// outputs false
System.out.println(myString == sameString);

// outputs true
System.out.println(myString.equals(sameString));

    Hint. By default, you can always use equals to compare the values of two strings. This is less error-prone than == (albeit potentially less efficient).
