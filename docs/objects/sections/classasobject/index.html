    Class as object #

    Some object-oriented languages (like Java, C# or C++) provide ways to access or manipulate a class (almost) as if it were an object.

    Static attribute and method #

    Static attribute #

    Terminology. Java (as well as C# and C++) supports attributes that do not belong to a specific instance, but to a class instead.

    These are often called static attributes (or sometimes class attributes or class variables), as opposed to the attributes that we have seen so far, called instance attributes (or member variables).

    Syntax. A static attribute is marked with the keyword static.

    Example. Consider a version of our game where default stats for each type of NPC (e.g. default XP) can be declared manually, in a text file. The path to this file could be stored as a static attribute, as follows:

    public class Unicorn {

    static String configFile = "units/unicorn.txt";

    int xp;

    public Unicorn(){
        xp = getXPFromConfig(configFile);
    }

    public learn(){
        xp += 4;
    }
}

    Warning. The value of a static attribute is unique (whereas for an instance attribute, each instance of the class carries its own value).

    Example (continued). The following creates in memory:

    • two values for xp (one carried by each object),
    • a unique string for configFile, which is carried by the class.
    Unicorn u1 = new Unicorn();
Unicorn u2 = new Unicorn();

    Static method #

    Similarly to a static attribute, a static method does not depend on a specific object (as opposed to an instance method).

    Example (continued). We can add to our class Unicorn a static method that checks whether the configuration file exists:

    public class Unicorn {
    ...

    static boolean configFileExists() {
        return Files.exists(Paths.get(configFile));
    }
}

    Note that the method getXPFromConfig (above) may also be static, if its execution is identical for all instances of Unicorn.

    Syntax. Recall that an instance method can be called outside of the class where it is declared, using an instance of the class followed by .. A static method can be called in the same way.

    Example.

    Unicorn u = new Unicorn();
boolean valid = u.configFileExists();

    Warning. This syntax is often discouraged, because it can be misleading (in this example, configFileExists can be mistaken for an instance method).

    Alternative syntax. Since the method depends on the class (rather than an instance), the call can be prefixed with the class name. This syntax is often preferred, because it leaves no ambiguity.

    Example.

    boolean valid = Unicorn.configFileExists();

    Restriction. There may be several instances of a same class in memory during the execution of a program (or none). This is why a static method cannot access an instance attribute or call an instance method.

    Example (continued). Adding the following static method to our class would cause a compilation error, because xp is an instance attribute.

    public class Unicorn {
    ...

    static void forget(){
        xp --;   // compilation error: `xp` is an instance attribute
    }
}

    Inheritance #

    Syntax. Two static methods with the same name and signature can be declared in a class and a superclass.

    Terminology. In Java, this is not called method overriding, but instead method hiding.

    Warning. Dynamic dispatch (a.k.a. runtime polymorphism) does not apply to this case: the method to be called is determined at compile time, rather than run time.

    Example (continued). Let us extend our example with a subclass of Unicorn that hides the static method configFileExists.

    public class EvilUnicorn extends Unicorn {

    static boolean configFileExists(){
        return false;
    }
}

    Now consider the following, where the method getUnicorns produces an array of unicorns that depend on the player’s input. The specific type of objects in this array (Unicorn or EvilUnicorn) cannot be determined at compile time.

    Unicorn[] unicorns = getUnicorns();
for (Unicorn unicorn: unicorns){
    System.out.println(unicorn.configFileExists());
}

    The method of the superclass (i.e. Unicorn.configFileExists) will be executed for each object in the array, regardless of its type.

    Warning. This is another reason why calling a static method via an instance (rather than via a class name) is often discouraged.

    Example (continued). Writing Unicorn.configFileExists() (with a capital U) would make it clear that the loop above serves no purpose (the same method configFileExists is executed unicorns.length times).

    Java interface #

    Since Java 8 (2014), an interface can also carry static methods and attributes.

    Reflection #

    Java (as well as C#) offers a mechanism called reflection to intuitively treat classes analogously to objects.

    Description. Each class of a program is associated with an object that represents it. This object is an instance of the Java class called Class.
    It can be accessed either:

    • via an instance of the class, or
    • via the qualified name of the class.

    Example. The object for the class Unicorn can be be accessed via an instance, as follows:

    Unicorn myUnicorn = new Unicorn();
Class unicornClass = myUnicorn.getClass();

    Alternatively, it can be accessed as follows (assuming that the file Unicorn.java is in the folder <workingDirectory>/src/main/java/org/npcs):

    Class unicornClass = Class.forName("org.npcs.Unicorn");

    Usage. The instance methods of the class Class are listed here. Among others, they allow:

    • listing the attributes of the class,
    • retrieving its immediate superclass, or the interfaces that it implements,
    • creating a new instance of this class,
    • etc.

    To go further. Reflection is a powerful feature. It is rarely used in everyday code, but can be helpful in specific situations. Notably, it is used (internally) by numerous Java frameworks such as Spring, Jackson or JUnit.
